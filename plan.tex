\section{Project Overview}

This project aims to adapt and re-implement the Agentic Graph RAG approach for vulnerability assessment, integrating cybersecurity knowledge graphs to rank vulnerabilities, generate risk assessments, and provide mitigation advice.

\subsection{Key Deliverables}
\begin{itemize}
    \item Agentic RAG system for vulnerability assessment
    \item Comparison with Normal RAG and Traditional CVSS scoring
    \item Evaluation results (accuracy, ranking quality, speed)
    \item 3 real vulnerability examples with system outputs
    \item GitHub repository with code and documentation
\end{itemize}

\subsection{Technical Stack}
\begin{itemize}
    \item \textbf{Knowledge Graph:} SEPSES Cybersecurity KG (CVE, CVSS, CWE, CAPEC)
    \item \textbf{Framework:} LangGraph for agent orchestration
    \item \textbf{Database:} Neo4j (LPG) + Virtuoso/SPARQL endpoint
    \item \textbf{LLM:} OpenAI GPT-4 or Claude (for prototyping)
    \item \textbf{Embedding:} all-MiniLM-L6-v2 (HuggingFace)
\end{itemize}

\section{Evaluation Metrics}

\subsection{Quantitative Metrics}
\begin{itemize}
    \item \textbf{Accuracy:} Precision, Recall, F1-score for vulnerability detection
    \item \textbf{Ranking Quality:} Spearman's correlation with ground truth CVSS
    \item \textbf{Speed:} Average response time per query (seconds)
    \item \textbf{Coverage:} \% of vulnerabilities correctly mapped to attack patterns
\end{itemize}

\subsection{Qualitative Assessment}
\begin{itemize}
    \item Clarity and actionability of risk assessments
    \item Relevance of mitigation recommendations
    \item Explanation quality and grounding in KG evidence
\end{itemize}

\section{Risk Mitigation}

\hline
SPARQL endpoint downtime & Cannot access SEPSES KG & Cache KG data locally, use backup endpoints \\
\hline
LLM API rate limits & Slow experimentation & Use local models (Llama, Mistral) as backup \\
\hline
Integration issues & Delays in testing & Weekly integration checkpoints, modular design \\
\hline
Insufficient dataset & Poor evaluation & Collect diverse CVEs early, validate with experts \\

\end{tabular}
